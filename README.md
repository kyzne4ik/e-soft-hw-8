# Домашнее задание: SQL — Основы работы с базой данных

**Зачем?**
Вы научитесь проектировать структуру данных для вашего проекта, описывать сущности с типами полей и выполнять базовые операции с базой данных через SQL. Это фундамент, без которого невозможно написать бэкенд, работающий с реальными данными.

---

## Что нужно сдать

1. **Файл `entities.md`** — описание всех сущностей вашего итогового проекта с типами полей.
2. **Скриншоты или логи** — подтверждение, что PostgreSQL установлен и работает.
3. **Скриншоты выполнения SQL-запросов** — результаты запуска готовых скриптов (CREATE TABLE, INSERT, SELECT).

## Часть 1 — Установка PostgreSQL

> **Цель:** развернуть PostgreSQL локально и убедиться, что к базе можно подключиться.

### Что нужно сделать (пошагово)

**Шаг 1. Скачайте PostgreSQL**

Перейдите на [официальный сайт](https://www.postgresql.org/download/) и скачайте версию для вашей ОС:

- **Windows:** скачайте интерактивный установщик с [EDB](https://www.enterprisedb.com/downloads/postgres-postgresql-downloads)
- **macOS:** `brew install postgresql@16` (через Homebrew)
- **Linux (Ubuntu/Debian):**
  ```bash
  sudo apt update
  sudo apt install postgresql postgresql-contrib
  ```

**Шаг 2. Установите**

При установке (Windows/macOS) вам предложат задать:
- **Пароль для суперпользователя `postgres`** — запомните его, он понадобится для подключения.
- **Порт** — оставьте по умолчанию `5432`.
- **Locale** — оставьте по умолчанию.

**Шаг 3. Проверьте установку**

Откройте терминал (или pgAdmin, если установился вместе с PostgreSQL) и выполните:

```bash
psql -U postgres
```

Введите пароль, который задали при установке. Если увидели приглашение `postgres=#` — всё работает.

Введите `\q` для выхода.

**Шаг 4. Создайте базу для практики**

```sql
CREATE DATABASE practice_db;
```

Подключитесь к ней:
```bash
psql -U postgres -d practice_db
```

### Что ожидается от студента (контрольные точки)

- [ ] `psql -U postgres` — подключение к БД работает
- [ ] Команда `SELECT version();` возвращает версию PostgreSQL
- [ ] База `practice_db` создана

## Часть 2 — Проектирование сущностей вашего проекта

> **Цель:** описать все сущности, которые будут в вашем итоговом проекте, и проставить типы каждого поля. Это ваш будущий DDL — структура таблиц.
>
> **Важно:** не нужно писать SQL-запросы. Опишите сущности на уровне схемы — как в примере ниже.

### Что нужно сделать

Откройте файл `entities.md` и опишите все сущности вашего проекта.

**Формат описания — для каждой сущности:**

```markdown
## Название сущности

Описание: что это за сущность и для чего нужна.

### Поля

| Поле        | Тип           | Обязательное | Описание                         |
|-------------|---------------|--------------|----------------------------------|
| id          | INTEGER       | Да           | Уникальный идентификатор (PK)    |
| title       | VARCHAR(200)  | Да           | Название, до 200 символов        |
| status      | VARCHAR(20)   | Да           | Статус, default: "active"        |
| created_at  | TIMESTAMP     | Да           | Дата создания, auto              |
| user_id     | INTEGER       | Да           | Ссылка на пользователя (FK)      |

```

### Подсказка по типам PostgreSQL

| Что храните           | Тип PostgreSQL       | Пример                    |
|-----------------------|----------------------|---------------------------|
| Целое число (ID)      | `INTEGER` / `BIGINT` | `42`                      |
| Строка до N символов  | `VARCHAR(N)`         | `"Иван"`                  |
| Длинный текст         | `TEXT`               | Статья, описание          |
| Да / Нет              | `BOOLEAN`            | `true`, `false`           |
| Дата и время          | `TIMESTAMP`          | `2026-05-19 14:30:00`     |
| Только дата           | `DATE`               | `2026-05-19`              |
| Десятичное число      | `NUMERIC(10,2)`      | `1999.99` (цена)          |
| JSON-данные           | `JSONB`              | `{ "key": "value" }`      |
| Перечисление          | `VARCHAR(20)`        | `"active"`, `"banned"`    |

### Пример: интернет-магазин

Если бы ваш проект был интернет-магазином, сущности могли бы выглядеть так:

**Users** — пользователи магазина:
- `id` INTEGER, PK
- `email` VARCHAR(255), уникальный
- `password_hash` VARCHAR(255)
- `role` VARCHAR(20), default: "customer"
- `created_at` TIMESTAMP

**Products** — товары:
- `id` INTEGER, PK
- `name` VARCHAR(200)
- `description` TEXT
- `price` NUMERIC(10,2)
- `in_stock` BOOLEAN, default: true
- `category_id` INTEGER, FK → Categories.id

**Orders** — заказы:
- `id` INTEGER, PK
- `user_id` INTEGER, FK → Users.id
- `status` VARCHAR(20), default: "created"
- `total_price` NUMERIC(10,2)
- `created_at` TIMESTAMP

**Описывать нужно именно ваши сущности для вашего итогового проекта**, а не этот пример.

- [ ] У каждой сущности есть `id` — первичный ключ
- [ ] Все поля имеют тип PostgreSQL (не `string`, не `number`, а `VARCHAR`, `INTEGER` и т.д.)
- [ ] Указаны обязательные / опциональные поля
- [ ] Описание каждой сущности — что это и зачем нужно

## Часть 3 — Первые SQL-запросы

> **Цель:** выполнить готовые SQL-скрипты и увидеть, как работают CREATE TABLE, INSERT и SELECT.
>
> **Скрипты** — В репозитории есть готовый .sql файлы. Вам нужно запустить их и разобраться, что делает каждая строка.

### Что нужно сделать (пошагово)

**Шаг 1. Подключитесь к вашей базе**

```bash
psql -U postgres -d practice_db
```

**Шаг 2. Выполните скрипт создания таблицы**

Преподаватель выдаст файл `01_create_table.sql`. Выполните его:

**Вариант А — через psql:**
```bash
\i 'путь/к/файлу/01_create_table.sql'
```

**Вариант Б — через консоль:**
```bash
psql -U postgres -d practice_db -f 01_create_table.sql
```

Убедитесь, что таблица создана:
```sql
\dt
```

Посмотрите структуру таблицы:
```sql
\d имя_таблицы
```

**Шаг 3. Выполните скрипт добавления данных**

Запустите файл `02_insert.sql` (так же, как в шаге 2).

Проверьте данные:
```sql
SELECT * FROM имя_таблицы;
```

**Шаг 4. Выполните скрипт выборки**

Запустите файл `03_select.sql`. Изучите каждый запрос в файле — какой результат он возвращает и почему.

**Шаг 5. Поэкспериментируйте**

Попробуйте выполнить несколько запросов самостоятельно:

```sql
-- Получить все записи
SELECT * FROM имя_таблицы;

-- Получить конкретные столбцы
SELECT name, price FROM имя_таблицы;

-- Фильтрация
SELECT * FROM имя_таблицы WHERE price > 100;

-- Сортировка
SELECT * FROM имя_таблицы ORDER BY price DESC;

-- Ограничение количества
SELECT * FROM имя_таблицы LIMIT 5;
```

- [ ] Данные добавлены через `02_insert.sql`
- [ ] SELECT-запросы из `03_select.sql` выполняются и возвращают результат
- [ ] Понимаете, что делает каждая строка в скриптах (преподаватель спросит)
- [ ] Выполнили хотя бы 3 запроса самостоятельно (не из скриптов)
