## User

Описание: Сущность пользователя, хранящая учётные данные, контактные данные и роли. Используется для аутентификации и авторизации.

### Поля

| Поле          | Тип       | Обязательное | Описание                                                             |
| ------------- | --------- | ------------ | -------------------------------------------------------------------- |
| id            | INTEGER   | Да           | Уникальный идентификатор (PK)                                        |
| first_name    | TEXT      | Да           | Имя пользователя                                                     |
| last_name     | TEXT      | Да           | Фамилия пользователя                                                 |
| patronymic    | TEXT      | Нет          | Отчество пользователя                                                |
| email         | TEXT      | Да           | Почта (UNIQUE, логин)                                                |
| password_hash | TEXT      | Да           | Хэш пароля                                                           |
| role          | TEXT      | Да           | Роль (ADMIN, MANAGER, MENTOR, STUDENT)                               |
| is_activated  | BOOLEAN   | Да           | Активация аккаунта (должен перейти по инвайт-ссылке), default: false |
| created_at    | TIMESTAMP | Да           | Дата создания, auto                                                  |
| updated_at    | TIMESTAMP | Да           | Дата обновления, auto                                                |

## Stream

Описание: Учебный поток (группа) - конкретная реализация курса с закреплённым менторов, датами старта и окончания. Например, "ШП-ВЕСНА-2026".
Хранит статус набора, активного обучения или завершён.

### Поля

| Поле       | Тип          | Обязательное | Описание                                  |
| ---------- | ------------ | ------------ | ----------------------------------------- |
| id         | INTEGER      | Да           | Уникальный идентификатор (PK)             |
| name       | TEXT         | Да           | Название потока (например, "ШП-2027")     |
| course_id  | INTEGER      | Да           | Ссылка на курс (FK)                       |
| mentor_id  | INTEGER      | Да           | Закрепленный ментор (FK к MentorProfile)  |
| status     | StreamStatus | Да           | Статус (ENROLLING, IN_PROGRESS, FINISHED) |
| created_at | TIMESTAMP    | Да           | Дата создания потока, auto                |
| updated_at | TIMESTAMP    | Да           | Дата обновления потока, auto              |

## **ENUM**: StreamStatus

Описание: Статус жизненного цикла учебного потока - от набора студентов до завершения.

### Поля

| Поле        |
| ----------- |
| ENROLLING   |
| IN_PROGRESS |
| FINISHED    |

## Вопросы для самопроверки

Напишите короткие ответы в `entities.md` в конце файла. Своими словами:

1. Почему важно указывать `WHERE` в `UPDATE` и `DELETE`?

**Ответ:** Потому что, произойдут изменения во всех строках таблицы. Это приведёт к массовому обновлению или удалению всех данных в таблице.

2. Что такое первичный ключ (PRIMARY KEY)? Зачем он нужен?

**Ответ:** Первичный ключ - это уникальный идентификатор каждой строки таблицы.

3. Приведите пример, когда `VARCHAR(20)` лучше чем `TEXT`, и наоборот.

**Ответ:** VARCHAR(20) - лучше, когда у вас не большая строка и заранее известно, что она будет маленькой (20 символов). TEXT - нужен только тогда, когда у вас очень длинная строка.
